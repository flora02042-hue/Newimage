# Inline Image Generation (Enhanced)

Расширение для SillyTavern. Ловит теги генерации в сообщениях ИИ и генерирует картинки через API.

**Enhanced версия** с дополнительными функциями для nano-banana:
- 🎨 Positive/Negative промпты
- 🖼️ Фиксированный стиль
- 👤 Автоматическое извлечение внешности из карточки персонажа
- 👕 Определение одежды из чата

## Формат тега

### Новый формат (рекомендуется)

```html
<img data-iig-instruction='{"style":"anime","prompt":"девушка с красными волосами"}' src="[IMG:GEN]">
```

После генерации расширение заменяет `src="[IMG:GEN]"` на реальный путь:
```html
<img data-iig-instruction='{"style":"anime","prompt":"..."}' src="/user/images/character/image.jpg">
```

LLM видит тот же формат, но понимает: есть реальный путь = уже сгенерировано.

### Legacy формат (поддерживается)

```
[IMG:GEN:{"style":"anime","prompt":"девушка с красными волосами"}]
```

### Параметры

| Параметр | Описание | Пример |
|----------|----------|--------|
| `style` | Стиль генерации (опционально) | `"anime"`, `"realistic"` |
| `prompt` | Описание картинки | `"девушка с красными волосами"` |
| `aspect_ratio` | Соотношение сторон | `"16:9"`, `"9:16"`, `"1:1"` |
| `image_size` | Разрешение (для nano-banana) | `"1K"`, `"2K"`, `"4K"` |
| `quality` | Качество (для OpenAI) | `"standard"`, `"hd"` |

## Настройки

Открыть Extensions → Генерация картинок

### Основные

- **Тип API**: OpenAI-совместимый или Gemini (nano-banana)
- **URL эндпоинта**: базовый URL API
- **API ключ**: ключ авторизации
- **Модель**: выбрать из списка (кнопка обновления подтягивает модели с `/v1/models`)
- **Размер**: 1024x1024, 1792x1024, 1024x1792, 512x512 (для OpenAI)
- **Качество**: standard / hd (для OpenAI)

### Для nano-banana

- **Соотношение сторон**: 1:1, 2:3, 3:2, 3:4, 4:3, 4:5, 5:4, 9:16, 16:9, 21:9
- **Разрешение**: 1K, 2K, 4K

### Референсы (nano-banana)

Отправка аватарок как референсов для консистентной генерации персонажей:

- **Отправлять аватар {{char}}** — автоматически берётся аватар текущего персонажа
- **Отправлять аватар {{user}}** — выбирается вручную из списка `/User Avatars/`

## 🆕 Новые функции (Enhanced)

### 🎨 Пользовательские промпты

**Positive промпт** — добавляется В НАЧАЛО каждой генерации, перед описанием персонажа.
Примеры:
- `masterpiece, best quality, detailed, sharp focus`
- `cinematic lighting, professional photography`
- `highly detailed anime style, vibrant colors`

**Negative промпт** — добавляется как инструкция избегания `[AVOID: ...]` в конец промпта.
Примеры:
- `low quality, blurry, deformed, ugly, bad anatomy`
- `watermark, signature, text, extra fingers`

### 🖼️ Фиксированный стиль

Когда включен, выбранный стиль будет применяться **ко ВСЕМ генерациям** и не будет меняться, независимо от стиля в теге.

Примеры стилей:
- `Avatar movie style, blue skin aliens, bioluminescent plants`
- `Anime Lycoris Recoil style, detailed, semi-realistic`
- `Cyberpunk 2077 game style, neon lights, futuristic city`
- `Studio Ghibli style, soft colors, detailed backgrounds`
- `Dark Souls style, gothic fantasy, dark atmosphere`

Стиль идёт **первым** в промпте как `[STYLE: ваш стиль]` для максимальной консистентности.

### 👤 Извлечение внешности

Автоматически парсит описание карточки персонажа и извлекает:
- Цвет и стиль волос
- Цвет глаз
- Цвет кожи
- Телосложение, рост
- Особые черты (уши, хвост, рога, крылья)

Добавляется в промпт как `[Character Reference: ...]`

Также поддерживается извлечение описания пользователя из persona (если настроено).

### 👕 Определение одежды

Сканирует последние N сообщений чата (настраивается, по умолчанию 5) и ищет упоминания одежды:
- "одет в белую рубашку" → добавит в промпт
- "переоделась в платье" → добавит в промпт
- "wearing a black jacket" → добавит в промпт

Добавляется как `[Current Clothing: ...]`

## Порядок построения промпта

Для nano-banana промпт строится в следующем порядке:

1. **[STYLE: фиксированный стиль]** (если включен)
2. **Positive промпт** из настроек
3. **Style из тега** (если нет фиксированного стиля)
4. **[Character Reference: внешность]** (если включено извлечение)
5. **[User Reference: внешность]** (если есть persona)
6. **[Current Clothing: одежда]** (если включено определение)
7. **Основной промпт из тега**
8. **[AVOID: negative промпт]** (если задан)

Плюс референсные изображения (аватары) отправляются отдельно.

## Поддерживаемые API

**OpenAI-совместимый** — `/v1/images/generations`
- DALL-E, Midjourney, Stable Diffusion, FLUX и прочие через различных провайдеров/прокси

**Gemini-совместимый** — `/v1beta/models/{model}:generateContent`
- Nano Banana/Nano Banana Pro через Google/прокси

## Как работает

1. ИИ пишет сообщение с тегом `<img data-iig-instruction='...' src="[IMG:GEN]">`
2. Расширение парсит тег, показывает спиннер
3. Собирает:
   - Референсы аватаров (если включены)
   - Внешность из карточки (если включено)
   - Одежду из чата (если включено)
   - Positive/Negative промпты
   - Фиксированный стиль (если включён)
4. Отправляет запрос на API
5. Заменяет `src="[IMG:GEN]"` на реальный путь к картинке
6. Сохраняет в чат
